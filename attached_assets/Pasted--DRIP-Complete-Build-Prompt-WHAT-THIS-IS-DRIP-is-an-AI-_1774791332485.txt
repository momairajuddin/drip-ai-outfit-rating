# DRIP. — Complete Build Prompt

## WHAT THIS IS

DRIP. is an AI-powered outfit rating app. You take a mirror selfie, AI scores your outfit 1-10 with a detailed breakdown, shows you similar outfits for inspiration, and recommends specific upgrades to raise your score. Think "CalAI for food, Umax for faces — DRIP. for fashion." The result is a beautiful, shareable score card designed to go viral on TikTok and Instagram.

This is a full-stack application: React frontend deployed as a Progressive Web App (installable on both iOS and Android home screens), Node/Express backend, SQLite database, and OpenAI Vision API integration. It must be publicly accessible via URL so anyone can use it from their phone.

---

## PLATFORM STRATEGY

Build as a **Progressive Web App (PWA)** with full mobile optimization. This is the fastest path to both iOS and Android:
- Users visit the URL on any phone browser
- "Add to Home Screen" prompt makes it feel native (full-screen, no browser chrome, app icon)
- Works identically on iOS Safari and Android Chrome
- No App Store approval needed — judges at the competition can try it instantly on their own phones
- Include a proper `manifest.json` and service worker for PWA installability

The entire UI must be designed mobile-first at 390px width. On desktop, center the app in a phone-shaped container (max-width 430px, centered, with a subtle device frame shadow) so it always looks like a mobile app.

---

## TECH STACK

- **Frontend:** React 18+ with Vite, TailwindCSS, Framer Motion for animations
- **Backend:** Node.js with Express
- **Database:** SQLite via better-sqlite3
- **AI:** OpenAI API (GPT-4o with vision) for outfit analysis
- **Image Handling:** Multer for upload, sharp for image processing/compression
- **Auth:** Simple JWT-based auth (email + password)
- **Deployment:** Replit auto-deploy (publicly accessible URL)
- **PWA:** manifest.json + service worker for home screen installation

Environment variables needed:
```
OPENAI_API_KEY=
JWT_SECRET=
```

## DESIGN SYSTEM — "Editorial Noir"

### Aesthetic Direction
The design language is **luxury fashion editorial meets minimalist tech**. Think: the visual identity of a Celine runway show crossed with Apple's hardware design. True black. Sharp white. Surgical typography. Every pixel is deliberate. The app should feel like picking up a Vogue magazine printed on obsidian glass.

This is NOT generic dark mode. This is fashion-world darkness — confident, editorial, elevated. The only color comes from the score itself, which shifts based on rating tier (gold for elite, silver for great, etc). This creates a natural psychological drive: users want to see GOLD.

### Typography
- **Display / Brand:** "Cormorant Garamond" (Google Fonts) — weight 600/700. This is a high-fashion serif used by actual luxury brands. Used for: the app name "DRIP.", score numbers, section titles, the result card headline. This font alone separates the app from every generic tech product.
- **Body / UI:** "Outfit" (Google Fonts) — weight 300/400/500. A modern geometric sans-serif. Clean, readable, slightly fashion-forward. Used for: body text, labels, buttons, descriptions.
- **Data / Scores:** "Cormorant Garamond" at large sizes with letter-spacing: -0.02em. Scores should feel like a magazine cover headline, not a dashboard metric.

### Layout Rules
- Max app width: 430px, centered on desktop inside a subtle phone-frame shadow
- All content padding: 20px horizontal
- Card border-radius: 2px (sharp, editorial — NOT rounded. Fashion is angular.)
- Button border-radius: 0px (fully sharp/square — this is a design choice that feels intentional and high-fashion)
- Card borders: 1px solid var(--border), no box-shadows (shadows feel techy, borders feel editorial)
- Section spacing: 48px between major sections
- Text should breathe — generous line-height (1.6 for body, 1.1 for display)
- ALL CAPS for labels and category names (letter-spacing: 0.12em) — editorial convention

### Motion & Animation (Framer Motion)
- Page transitions: fade + subtle vertical shift (y: 10px → 0), duration 0.3s
- Cards: stagger fade-in, 80ms delay between each
- Score reveal: the number should "count up" from 0 to final score over 1.5 seconds with an easing curve, then pulse-glow once in its tier color
- Photo upload: subtle scale animation on the preview
- Pull-to-refresh: custom animation (not browser default)
- Skeleton loading: dark shimmer effect (from #0A0A0A to #151515, not grey)
- Button press: scale(0.97) for 100ms, no color change — tactile, minimal
- Score card entrance: cinematic — fade up from black with slight blur-to-clear transition

### Photography & Image Treatment
- User photos displayed with a very subtle vignette overlay (darker edges)
- All outfit photos (user's and similar outfits) displayed in consistent aspect ratios — 3:4 portrait
- Similar outfit images should feel like an editorial mood board grid

## BACKEND API DESIGN

### Endpoints

```
POST   /api/auth/register              — Create account (email, password, username)
POST   /api/auth/login                 — Login, returns JWT
GET    /api/auth/me                    — Get current user from token

POST   /api/scan                       — Upload outfit photo, receive AI analysis
GET    /api/scan/history               — Get user's scan history (paginated)
GET    /api/scan/:id                   — Get single scan result with full details

GET    /api/style-dna                  — Get user's Style DNA profile (aggregated from scans)
GET    /api/style-dna/report           — Get monthly Style Report data

GET    /api/similar/:scanId            — Get similar outfit inspiration for a scan
GET    /api/recommendations/:scanId    — Get upgrade recommendations for a scan

GET    /api/leaderboard                — Top scores (anonymous, by score)
GET    /api/stats                      — Global app stats for landing page
```

### Core AI Integration — The /api/scan Endpoint

When a user uploads an outfit photo, the backend:

1. Saves the image, compresses it with sharp
2. Sends it to OpenAI GPT-4o Vision API with this system prompt:

```
You are DRIP., the world's most sophisticated AI fashion analyst. You combine the eye of a Vogue editor, the precision of a stylist, and the cultural awareness of a trend forecaster.

Analyze this outfit photo and return a comprehensive rating. Be specific, opinionated, and constructive — never generic. Reference actual fashion concepts, color theory, and current trends.

Return ONLY valid JSON — no markdown, no backticks, no explanation outside the JSON:

{
  "overall_score": 8.4,
  "scores": {
    "fit": {
      "score": 9,
      "detail": "Specific observation about how garments fit the body — proportions, tailoring, silhouette"
    },
    "color": {
      "score": 7,
      "detail": "Analysis of color palette, harmony, contrast, skin tone compatibility"
    },
    "style_coherence": {
      "score": 8,
      "detail": "How well the pieces work together as a unified look — does it tell a story?"
    },
    "trend_alignment": {
      "score": 9,
      "detail": "How current/relevant the outfit feels in 2026 fashion landscape"
    },
    "signature_factor": {
      "score": 7,
      "detail": "Does this outfit have personality? Is it memorable or forgettable?"
    }
  },
  "style_archetype": "Old Money Casual",
  "style_archetype_description": "1-2 sentence description of this archetype",
  "vibe_tags": ["effortless", "refined", "understated"],
  "best_element": "The oversized blazer creates a strong shoulder line that anchors the whole look",
  "weakest_element": "The brown shoes clash with the cool-toned palette — a tonal mismatch",
  "occasion_match": {
    "casual_hangout": 9,
    "date_night": 7,
    "work_office": 8,
    "formal_event": 5,
    "streetwear_flex": 6
  },
  "upgrades": [
    {
      "swap": "Replace brown leather shoes",
      "with": "White leather minimal sneakers or black chelsea boots",
      "predicted_score_increase": 0.7,
      "search_query": "white minimal leather sneakers men",
      "price_range": "$80-$150"
    },
    {
      "swap": "Add an accessory",
      "with": "Silver minimal watch or thin chain necklace",
      "predicted_score_increase": 0.4,
      "search_query": "minimal silver watch",
      "price_range": "$40-$120"
    }
  ],
  "similar_style_search_queries": [
    "old money casual men fall outfit",
    "blazer with relaxed trousers editorial",
    "quiet luxury menswear 2026"
  ],
  "color_palette_detected": ["#2C3E50", "#ECF0F1", "#8B4513"],
  "season_alignment": "fall/winter",
  "celebrity_style_match": {
    "name": "Jacob Elordi",
    "match_percentage": 78,
    "description": "Shares the tall, relaxed tailoring sensibility"
  },
  "gender_detected": "male"
}

RULES:
- Be brutally honest but never cruel. Think constructive fashion editor, not mean judge.
- Overall score should be an average weighted toward fit (25%), color (20%), style_coherence (25%), trend_alignment (15%), signature_factor (15%).
- Scores 9-10 should be RARE and genuinely earned. Most outfits are 5-8. Don't inflate.
- If the image is not a person wearing clothes (e.g., a landscape, food, object), return: {"error": "no_outfit_detected", "message": "I need a photo of you wearing an outfit to rate your drip."}
- Always provide at least 2 upgrade suggestions.
- similar_style_search_queries should be specific enough to find similar outfits on Google/Pinterest.
- Detect gender from the outfit to tailor advice appropriately. Use neutral language if ambiguous.
```

3. Parses the JSON response
4. Stores the scan result in the database
5. Asynchronously fetches similar outfit images using the similar_style_search_queries (use a Google Custom Search API or pre-loaded curated outfit database)
6. Returns the complete result to the frontend

### Fallback System
If the OpenAI API fails or times out (>15 seconds), return a pre-built fallback analysis with realistic scores and generic-but-good advice. The demo must NEVER break. Store 5-6 different fallback results and randomly select one, adjusting the style archetype based on simple image characteristics.

## DATABASE SCHEMA

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT UNIQUE NOT NULL,
  username TEXT UNIQUE NOT NULL,
  password_hash TEXT NOT NULL,
  is_premium BOOLEAN DEFAULT FALSE,
  total_scans INTEGER DEFAULT 0,
  highest_score REAL DEFAULT 0,
  avg_score REAL DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE scans (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER REFERENCES users(id),
  image_path TEXT NOT NULL,
  overall_score REAL NOT NULL,
  fit_score REAL,
  color_score REAL,
  coherence_score REAL,
  trend_score REAL,
  signature_score REAL,
  style_archetype TEXT,
  archetype_description TEXT,
  vibe_tags TEXT,              -- JSON array
  best_element TEXT,
  weakest_element TEXT,
  occasion_match TEXT,         -- JSON object
  upgrades TEXT,               -- JSON array
  similar_queries TEXT,        -- JSON array
  color_palette TEXT,          -- JSON array
  season TEXT,
  celebrity_match TEXT,        -- JSON object
  gender_detected TEXT,
  full_analysis TEXT,          -- Full raw JSON from AI
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE similar_outfits (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  scan_id INTEGER REFERENCES scans(id),
  image_url TEXT NOT NULL,
  source_url TEXT,
  title TEXT,
  style_tag TEXT
);

CREATE TABLE style_dna (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER REFERENCES users(id) UNIQUE,
  dominant_archetype TEXT,
  top_colors TEXT,             -- JSON array of most-worn hex colors
  avg_scores TEXT,             -- JSON object of category averages
  total_outfits_scanned INTEGER DEFAULT 0,
  best_outfit_id INTEGER REFERENCES scans(id),
  worst_outfit_id INTEGER REFERENCES scans(id),
  style_evolution TEXT,        -- JSON array of archetype over time
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## SIMILAR OUTFITS SYSTEM

For the "Similar Drip" section on each result, implement this:

**Option A (preferred if API available):** Use Google Custom Search API (or Bing Image Search API) with the AI-generated search queries to fetch real outfit images from Pinterest, fashion blogs, etc.

**Option B (fallback — always implement this):** Pre-load a curated database of 100-200 outfit images organized by style archetype. Store them in a `curated_outfits` table:

```sql
CREATE TABLE curated_outfits (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  image_url TEXT NOT NULL,
  style_archetype TEXT NOT NULL,
  gender TEXT,
  season TEXT,
  tags TEXT,  -- JSON array: ["casual", "streetwear", "oversized"]
  source_credit TEXT
);
```

Seed this with real outfit inspiration images from free fashion image APIs or public Pinterest boards. Match them to the user's scan based on style_archetype and gender_detected.

## ANIMATIONS SPECIFICATION

### Score Counter Reveal
```
- Start at 0.0
- Count up to final score over 1.5 seconds
- Easing: cubic-bezier(0.16, 1, 0.3, 1) — fast start, smooth deceleration
- At completion: single pulse-glow in tier color (scale 1.0 → 1.05 → 1.0, opacity of glow 0 → 0.3 → 0)
- Score font: Cormorant Garamond, 72px
```

### Category Bars
```
- Each bar animates from width 0% to final percentage
- Duration: 0.8s per bar
- Stagger: 100ms between bars
- Easing: ease-out
```

### Scanning Screen
```
- Scan line: white horizontal line, 1px, 40% opacity
- Moves top-to-bottom over 2 seconds, repeats
- Ken Burns on photo: scale 1.0 → 1.05 over 10 seconds
- Status text: crossfade transition, 2s per message
```

### Share Card Generation

## GLOBAL APP BEHAVIOR

### Navigation
- Bottom tab bar with 3 tabs:
  1. **Camera** (home icon or viewfinder icon) — the scan page
  2. **Archive** (grid icon) — scan history
  3. **DNA** (fingerprint or DNA helix icon) — Style DNA page
- Current tab indicated by white icon, inactive tabs in grey-3
- Tab bar: black background, thin top border in var(--border)
- No tab labels — icons only, minimal

### Haptic Feedback (where supported)
- Trigger navigator.vibrate() on:
  - Score reveal moment (short buzz)
  - Successful share (short buzz)
  - Tab switches (micro buzz)

### Image Upload Handling
- Accept camera capture and gallery upload
- Compress images to max 1200px on longest edge before upload (saves bandwidth)
- Show upload progress bar during transfer
- Max file size: 10MB
- Accepted formats: jpg, png, webp, heic (convert heic to jpg server-side)

### Error States
- API timeout: show fallback results, never a blank screen
- No internet: show cached last scan, "Offline" banner at top
- Invalid image (not an outfit): show the AI's error message in a clean card with "Try Again" button
- All errors use the same black card style — no jarring red/white error modals

### Performance
- Lazy load images below the fold
- Skeleton screens for all loading states (dark shimmer, not grey)
- Debounce rapid scan button taps (prevent double-submission)
- Cache the last 3 scan results client-side for instant re-viewing
